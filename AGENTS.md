# AGENTS.md

This file provides repository guidance for AI coding agents.

## Stack

See `STACK.md` for runtime, commands, and environment.

## Required Context

Before making changes, read:

- `PROJECT.md` - project purpose, architecture, risks, and priorities
- `STACK.md` - runtime, commands, tests, security scans, and environment notes
- `README.md` - user-facing behavior and setup
- `CHANGELOG.md` - recent notable changes
- `CONTRIBUTING.md` - development and validation workflow
- `SECURITY.md` - vulnerability reporting and sensitive-data handling

If a local `AGENT.md` file exists, agents should also read it for local workflow preferences. Do not rely on `AGENT.md` being present.

## Core Rules

- Prefer small, reviewable changes.
- Preserve existing behavior unless explicitly asked to change it.
- Follow the commands and environment policy in `STACK.md`.
- Use the project-local virtual environment when running Python commands.
- Do not install dependencies globally.
- Do not commit secrets, tokens, local config, runtime artifacts, private security reports, or local agent-control files.
- Do not rewrite unrelated code.
- Do not reformat unrelated files.
- Do not amend, rebase, force-push, or switch branches unless explicitly requested.
- Run the narrowest relevant validation after code changes.
- Do not claim tests passed unless they were actually run.
- Report what changed, what was verified, and any remaining risk.

## Security

Treat these as high-risk areas:

- listener binding and public share behavior
- authentication and authorization
- token or credential handling
- TLS verification
- model downloads
- file path handling
- prompt template paths
- logging of sensitive data
- dependency changes

For high-risk changes, keep the change minimal, add or update tests where practical, and run the relevant security checks from `STACK.md`.

## Documentation

Update documentation only when relevant:

- `README.md` for user-facing behavior or setup changes
- `CHANGELOG.md` for notable changes
- `CONTRIBUTING.md` for development workflow changes
- `PROJECT.md` or `STACK.md` for architecture, command, or environment changes

Do not edit this file unless explicitly requested.
